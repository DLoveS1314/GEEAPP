## 3. View-Based DEM Loading (Default to DEM)

### Files to Modify
- `backend/src/routes/gee.js` - Add view-based DEM endpoint
- `backend/src/services/geeService.js` - Add view extent DEM sampling
- `front/src/App.vue` - Set DEM as default data source
- `front/src/useMap.js` - Update default view initialization

### Implementation Details
1. Backend:
   - Add POST `/api/gee/dem/sample` endpoint (already exists, enhance it)
   - Accept viewport bounds: `{bounds: [minLon, minLat, maxLon, maxLat]}`
   - Sample DEM data only within visible extent
   - Return sampled hexagons with DEM values

2. Frontend:
   - Set `dem-srtm` as default data source on initial load
   - Keep an initial view only as a fallback viewport, but no backend GEE layer should be clipped to Beijing
   - When loading DEM or sampling, send current map view extent instead of a fixed Beijing center point

3. Logic Changes:
   - Default to DEM data source selection on startup
   - When user zooms/pans, DEM data can be refreshed for new extent

## Detailed Follow-Up Implementation Checklist

### A. Backend Datasource Configuration

1. Complete `backend/src/services/geeService.js` datasource loader.
   - Add `loadDatasourceConfig()` to read `../config/datasources.json` with `fs.readFile`.
   - Cache parsed config in `datasourcesCache`.
   - Validate that `dataSources` is an array and each entry has `id`, `name`, `type`, `description`, `dataset`, and `visualization`.
   - Return friendly 500 errors for invalid JSON or missing required fields.

2. Replace hard-coded layer catalog.
   - Change `getLayerCatalog()` to return sanitized datasource entries from JSON.
   - Keep fields needed by frontend: `id`, `name`, `type`, `description`, `dataset`, `defaultCenter`, `defaultZoom`, `bounds`.

3. Replace hard-coded layer selection in `getLayerById()`.
   - Lookup datasource by `layerId` from JSON.
   - Dispatch by datasource `type` or `id` to image builders.
   - Use datasource `visualization` object instead of hard-coded visualization params.

4. Add image builders.
   - `RGB`: Sentinel-2 median composite without clipping to Beijing by default; if a viewport bbox is passed later, clip/filter by that bbox.
   - `LANDCOVER`: ESA WorldCover image, with correct class palette colors rather than class labels as palette values.
   - `DEM`: SRTM DEM image.

5. Add service methods for new API needs.
   - `getDatasourceCatalog()` can alias `getLayerCatalog()` or return `{ dataSources }`.
   - `sampleDemForHexagons(geojson, options)` for center-point DEM sampling.
   - `filterGeojsonByBounds(geojson, bounds)` for extracting only visible hexagons from L4 when requested.

### B. Backend Routes

1. Update `backend/src/routes/gee.js` imports.
   - Import datasource catalog and DEM sampling service functions.

2. Add `GET /api/gee/datasources`.
   - Return `{ dataSources }` loaded from JSON.
   - This can share the same payload shape as `/layers`, but should make the JSON-source behavior explicit.

3. Add `GET /api/gee/geojson/hexagons`.
   - Read `backend/data/L4.geojson`.
   - Optional query params: `minLon`, `minLat`, `maxLon`, `maxLat`, `limit`.
   - For initial implementation, allow returning a bounded subset when bounds are provided, and reject unbounded full L4 if it is too large.
   - Response type: `application/geo+json`.

4. Add or complete `POST /api/gee/dem/sample`.
   - Request body should support `{ bounds, geojson, projection, save }`.
   - Prefer `geojson` if provided, otherwise read L4 and filter by `bounds`.
   - Sample DEM at each hexagon center point.
   - Return `{ hexagons, sampledCount, savedPath? }`.

5. Optional save behavior.
   - If `save: true`, write sampled output to `backend/output/dem-sampled-<timestamp>.geojson`.
   - If `save` is false or omitted, keep sampling in memory only.

### C. Frontend API Client

1. Extend `front/src/api.js`.
   - Add `fetchDataSources()` for `/api/gee/datasources`.
   - Add `fetchHexagonGeojson(params)` for `/api/gee/geojson/hexagons`.
   - Add `sampleDem(payload)` wrapper for `POST /api/gee/dem/sample`.
   - Keep centralized error handling in `request()`; extend it with optional method/body support rather than duplicating raw `fetch()` calls.

### D. Frontend Map Composable

1. Remove OSM.
   - Remove `OSM` import.
   - Remove `baseLayer` variable.
   - Build OpenLayers map with `[geeLayer, demHexLayer]` only.
   - Remove `base` branch from `updateLayerVisibility()`.

2. Add viewport helpers.
   - `getViewBounds()` should return `[minLon, minLat, maxLon, maxLat]` converted from current EPSG:3857 extent.
   - `setDemHexagons(geojson, config, options)` should support `fit: false` so viewport-based refresh does not always zoom away.

3. Add style distinction.
   - Keep DEM sampled hexagons as elevation color ramp.
   - Add `setRawHexagons()` or a style mode option for loaded raw L4 hexagons with transparent fill and clear stroke.

### E. Frontend App Workflow

1. Data source dropdown.
   - Replace “示例图层” with “GEE 数据源”.
   - Populate from `fetchDataSources()` or `fetchLayers()` backed by JSON.
   - Default selected ID should be `dem-srtm` when present.
   - Display selected datasource description in metadata panel.

2. Default DEM loading.
   - On bootstrap, load `dem-srtm` layer first.
   - Do not fetch precomputed L0 DEM by default unless needed as fallback.
   - Use current viewport bounds for DEM sampling/rendering where applicable.

3. Add buttons.
   - “加载六角格”：fetch L4 hexagons from backend using current viewport bounds.
   - “采样 DEM”：send currently loaded hexagons or current viewport bounds to sampling endpoint.
   - “导出 DEM”：keep existing browser-side export for current sampled GeoJSON.
   - Optional “采样并保存”：send `save: true` to backend and display returned `savedPath`.

4. State handling.
   - Track `hexagonGeojson` separately from `demGeojson` so raw L4 and sampled DEM are not confused.
   - Disable sampling until raw hexagons are loaded or viewport bounds are available.
   - Reset `rawDataVisible` consistently after new sampled DEM is applied.
